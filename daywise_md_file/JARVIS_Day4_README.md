# JARVIS MARK-I

# Day 4 — Continuous Assistant Mode

## Overview
Day 4 transformed Jarvis from a one-command script into a persistent assistant.

## Features Implemented
- Continuous listening loop
- Exit command
- Sleep mode
- Wake mode
- State management
- Error handling

## Concepts Learned

### Infinite Loop
```python
while True:
```
Keeps Jarvis running forever.

### break
```python
break
```
Stops the loop.

### continue
```python
continue
```
Skips current iteration.

### State Management
```python
jarvis_awake = True
```
Controls assistant mode.

## Workflow

START
↓
Greeting
↓
Listen
↓
Recognize
↓
Check State
↓
Execute Command
↓
Respond
↓
Listen Again

## Day 4 Achievement

Jarvis can now stay active, process multiple commands, sleep, wake, and exit gracefully.
